# Patch Checklist

本清单用于配合 `create-nsis-frontend-patch-installer` Skill 使用。  
目标是保证“前端补丁包”只在真正适合的场景下使用，并且生成的 NSIS 补丁包能够安全升级已安装机器，而不是误伤程序层或数据层。

## 1. 适用性判断

- [ ] 本次变更只涉及前端静态资源或前端逻辑
- [ ] 本次没有后端接口变更
- [ ] 本次没有数据库 schema 变化
- [ ] 本次没有 runtime 变化
- [ ] 本次没有桌面壳变化
- [ ] 本次没有 launcher 变化

若以上任一项不满足，应重新评估是否必须改走全量安装包流程。

## 2. 输入确认

- [ ] 已确认打包工作区目录
- [ ] 已确认 `package-root/app/frontend` 已准备完成
- [ ] 已确认 NSIS 可执行路径可用
- [ ] 已确认当前版本号
- [ ] 已确认补丁输出目录
- [ ] 已确认别名文件名策略

## 3. 安装目标识别

- [ ] 安装器优先从历史 `InstallLocation` 识别旧安装目录
- [ ] 若历史注册信息缺失，已定义人工确认策略
- [ ] 已确认补丁包不承担首装职责
- [ ] 已确认用户若未安装，需要改用全量安装包

## 4. 升级前停栈

- [ ] 已设计停桌面壳逻辑
- [ ] 已设计停后台脚本调用逻辑
- [ ] 已定义 PID 兜底策略
- [ ] 已定义端口探测或健康检查兜底策略
- [ ] 停栈失败时不会静默继续覆盖

## 5. 备份策略

- [ ] 已定义旧版 `app/frontend` 备份路径
- [ ] 备份路径包含时间戳
- [ ] 备份行为会写入安装日志
- [ ] 已确认不会误删备份目录

## 6. 覆盖边界

- [ ] 仅删除/覆盖 `app/frontend`
- [ ] 不覆盖 `app/backend`
- [ ] 不覆盖 `app/runtime`
- [ ] 不覆盖 `app/desktop`
- [ ] 不触碰 `data/`

## 7. 版本标记

- [ ] 已确认补丁版本号
- [ ] 已确认安装器展示版本号
- [ ] 已确认必要注册表版本字段更新策略
- [ ] 已确认别名包命名规则

## 8. 构建结果

- [ ] NSIS 补丁安装器构建成功
- [ ] 主补丁包已生成
- [ ] 别名包已生成
- [ ] `sha256` 文件已生成
- [ ] 产物命名符合当前版本规则

## 9. 最小验收

- [ ] 已安装版本可识别并进入升级流程
- [ ] 升级前旧前端已备份
- [ ] 升级后前端页面成功更新
- [ ] 升级后数据目录未受影响
- [ ] 升级后后端仍可正常联通
- [ ] 升级后桌面壳仍可正常启动

## 10. 交付输出

- [ ] 已记录补丁覆盖范围
- [ ] 已记录保留目录
- [ ] 已记录停栈方式
- [ ] 已记录备份位置
- [ ] 已记录主补丁包路径
- [ ] 已记录别名包路径
- [ ] 已记录 `sha256` 路径

## 11. 高风险误用提醒

出现以下任一情况，不应继续把它当成前端补丁包处理：

- [ ] 后端接口返回结构已变化
- [ ] 数据库迁移已变化
- [ ] launcher 启停脚本已变化
- [ ] 桌面壳导航方式已变化
- [ ] runtime 版本已变化

如果命中上述任一项，应优先考虑全量安装包或更完整的程序层升级方案。

## 12. 最终完成定义

只有在以下条件同时满足时，才算这个 Skill 真正完成：

- 适用性判断通过
- 升级前停栈逻辑明确
- 旧前端已备份
- 仅覆盖 `app/frontend`
- `data/` 保持不动
- 补丁包与校验文件生成成功
- 最小验收结果已明确
