# Build Checklist

本清单用于配合 `build-windows-offline-desktop-package` Skill 使用，帮助执行者在真正开始 Windows 离线桌面打包前后，按固定顺序完成检查、组装、构建、验收和交付。

## 1. 输入确认

- [ ] 已确认主源码目录
- [ ] 已确认打包工作区目录
- [ ] 已确认当前产品版本号
- [ ] 已确认本次需要的是全量安装包，而不是前端补丁包
- [ ] 已确认 NSIS 可执行路径可用
- [ ] 已确认图标资源已准备
- [ ] 已确认 Python 离线运行时资源已准备
- [ ] 已确认 WebView2 离线资源或前置条件已明确

## 2. 边界确认

- [ ] 主源码目录不作为打包实验目录使用
- [ ] 打包相关改动限定在独立 `_DEV` 或等效工作区
- [ ] 已明确哪些文件来自源码仓
- [ ] 已明确哪些文件是打包侧热修
- [ ] 已确认 `app/` / `data/` 分层模型不被破坏

## 3. 打包前清理

- [ ] 已清理无关旧版安装包产物
- [ ] 已清理会误导排查的历史构建缓存
- [ ] 已清理无关临时目录
- [ ] 已确认当前要保留的 `dist/`、`build/`、`package-root/` 目录

## 4. 源码同步

- [ ] 已将主源码同步到打包工作区
- [ ] 已同步必要的前端资源文件
- [ ] 已同步必要的后端脚本或配置
- [ ] 已同步桌面壳所需的入口文件
- [ ] 已记录本次同步来源或版本节点

## 5. 应用层组装

- [ ] 已成功重建 `package-root/app`
- [ ] 后端代码已复制到程序层
- [ ] 前端静态构建产物已复制到程序层
- [ ] launcher 脚本已落到程序层
- [ ] 桌面壳产物已进入程序层
- [ ] 运行时目录已进入程序层
- [ ] 程序层中不存在本不应保留的开发缓存

## 6. 运行时装配

- [ ] Python 运行时已装配完成
- [ ] Python 路径策略已检查
- [ ] WebView2 相关资源或前置条件已明确
- [ ] 本地后端可在打包结构下被正常启动
- [ ] 数据目录初始化路径已检查

## 7. 前端检查

- [ ] 前端已成功构建
- [ ] 前端构建结果可脱离开发服务器运行
- [ ] 路由模式适配本地桌面场景
- [ ] 资源路径不是错误的绝对路径
- [ ] 若使用 WebView2，本地入口地址策略已明确

## 8. 安装器构建

- [ ] 全量安装器脚本存在
- [ ] 安装器版本号正确
- [ ] 安装器图标正确
- [ ] 默认安装路径策略正确
- [ ] 安装器允许用户修改安装根目录
- [ ] 升级识别旧安装目录的逻辑已存在
- [ ] 升级前停栈逻辑已存在
- [ ] `setup.exe` 构建成功
- [ ] `sha256` 文件已生成
- [ ] IT 友好别名包已生成

## 9. 验收检查

- [ ] 可执行首次安装
- [ ] 首次启动成功
- [ ] 二次启动成功
- [ ] 桌面快捷方式可正常启动程序
- [ ] 安装后前后端功能链路可正常运行
- [ ] 数据目录可写
- [ ] 日志目录可写
- [ ] 卸载行为符合预期
- [ ] 升级覆盖策略符合预期

## 10. 交付输出

- [ ] 已记录安装包路径
- [ ] 已记录校验文件路径
- [ ] 已记录版本号
- [ ] 已记录输入来源
- [ ] 已记录已知限制
- [ ] 已记录推荐验收方式

## 11. 出错时先查什么

若构建失败，优先检查：

1. 打包工作区路径是否正确
2. 运行时资源是否缺失
3. 前端是否构建失败
4. 桌面壳是否发布失败
5. 安装器脚本是否存在编码或语法问题
6. 历史缓存是否干扰当前装配

## 12. 最终完成定义

只有在以下条件同时满足时，才算本 Skill 完成：

- `package-root/app` 已组装完成
- 全量安装包已生成
- 校验文件已生成
- 产物路径已明确
- 最小验收建议已给出
