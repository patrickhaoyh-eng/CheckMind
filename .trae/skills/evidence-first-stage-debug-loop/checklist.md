# 阶段性测试/调试检查清单

## 开始前
- 本轮目标是否只有一个最小验证点
- 成功信号是否明确
- 失败信号是否明确
- 证据目录或 run 目录是否已知
- 用户是否只保留必要交互

## 执行中
- 是否同步记录日志/marker/截图
- 是否记录关键环境变量或配置
- 是否保留失败现场
- 是否避免让用户代替 Agent 做重复动作

## 结束后
- 是否给出通过/失败/待补证据结论
- 是否列出证据路径
- 是否写清下一步最小动作
- 若连续两轮未定位，是否切换到插桩模式

## 禁止项
- 没有证据就说“已验证”
- 没有新增信息就再试一次
- 一轮里同时验证多个大目标
- 把可自动执行的测试动作推给用户
