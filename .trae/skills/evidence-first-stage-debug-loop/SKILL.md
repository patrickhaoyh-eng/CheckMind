# Evidence-First Stage Debug Loop

这个 Skill 用于在“实现完成后验证”“运行时问题定位”“需要用户参与但希望 Agent 带跑”的场景下，执行一套证据优先、阶段清晰、避免原地打转的测试/调试循环。

它的重点不是“多跑几次”，而是：

- 每轮先定义目标
- 每轮同步收集证据
- 每轮结束形成结论
- 连续失败后停止猜测，转向插桩和缩小边界
- 默认由 Agent 主导执行，尽量减少用户重复手工操作

## 角色定位

这个 Skill 是对现有项目规则和开发/测试约束的补充，不是替代。

- 它负责补上“阶段性测试/调试如何执行”的操作层空白。
- 它不能覆盖 `alwaysApply` 的项目护栏、证据规则、质量门禁或已有文档基线。
- 当本 Skill 与项目规则、顶层文档或已确认的任务计划冲突时，优先遵循上位约束，而不是改写它们。
- 新增这个 Skill 的目的，是以“打补丁堵漏洞”的方式补强流程，而不是抛弃旧技能、重建一套平行体系。

## 何时调用

在以下场景调用：

- 用户要求进入阶段性功能测试
- 用户要求留证据、做闭环验证、形成测试/调试记录
- 问题只靠静态代码阅读无法确认，需要运行时证据
- 需要用户参与确认，但希望 Agent 主导执行测试
- 已经连续几轮修了又试、试了又猜，需要停止发散

不适合以下场景：

- 单纯解释代码
- 纯静态文档修改且不需要验证
- 极小的、无需运行时证据的一次性改动

## 核心目标

1. 把当前问题收敛成一个最小测试目标
2. 明确成功信号、失败信号和证据要求
3. 由 Agent 发起测试、读取结果并给出结论
4. 若失败，基于证据生成最小下一步
5. 若两轮无法定位，自动切换到插桩模式

## 必须遵守的约束

- 不得宣称“已验证/已修复”，除非有证据
- 不得要求用户执行 Agent 可以自己完成的重复测试动作
- 不得在没有新增证据的情况下反复猜测
- 单个问题最多三轮“改代码后重测”；第三轮仍未定位必须升级处理
- 结论必须引用具体路径、日志、截图或结构化文件

## 推荐输入信息

调用前尽量确认：

- 当前要验证的功能或问题
- 本轮最小目标
- 运行入口
- 证据目录或 run 目录位置
- 用户是否需要参与确认
- 当前已知失败现象

如果以上信息缺失，先补齐，再执行。

## 标准执行步骤

### 第一步：收敛本轮目标
- 用一句话写清本轮要证明什么
- 明确成功信号和失败信号
- 明确本轮需要哪些最小证据

### 第二步：确认前置条件
- 检查环境变量、配置、窗口状态、样例数据、运行入口
- 若有用户交互，提前说明用户只需要做哪一步

### 第三步：由 Agent 发起执行
- 构建
- 运行
- 读取 run 目录、marker、error 文件、日志、截图
- 汇总现场现象

### 第四步：基于证据裁决
- 通过：证据满足验收信号
- 失败：证据与预期矛盾或缺少关键产物
- 待补证据：当前信息不足，不允许强行判断

### 第五步：失败后收敛最小下一步
- 写清事实
- 写清最小假设
- 写清最小插桩或检查动作

### 第六步：必要时升级
- 若连续两轮未定位，停止猜测
- 改为加 phase marker、结构化日志、输入输出快照
- 若仍无法收敛，拆分任务并明确阻塞原因

## 推荐输出格式

完成一轮后，至少输出：

```markdown
## 本轮目标
- 

## 实际结果
- 

## 证据路径
- 

## 当前结论
- 通过 / 失败 / 待补证据

## 下一步最小动作
- 
```

## 与用户协作的默认模式

- 默认 Agent 主导：
  - 运行命令
  - 检查文件
  - 汇总结论
- 默认用户最小参与：
  - 点击确认弹窗
  - 恢复窗口到前台
  - 提供现场不可替代反馈

若用户需要参与，必须明确告诉用户：

- 只需要做什么
- 什么时候做
- 成功时会看到什么
- 失败时需要回传什么证据

## 与规则/文档的关系

- 本 Skill 是执行型补充，不是总管型替代；它用于落实已有规则，不用于废弃已有规则。
- 若工作区已有测试/调试执行文档，优先遵循该文档
- 在 CheckMind 项目中，默认参考：
  - `docs/05-阶段性测试与调试执行规范.md`
  - `testing-evidence-collector`
  - `testing-stage-debug-governor`

## 示例

### 示例 1：真实机抓取卡点
- 用户说：“开始弹窗后就卡住了，帮我带跑测试并定位卡点”
- 你应调用本 Skill

### 示例 2：需要用户确认但不想全程手测
- 用户说：“你来执行大部分测试，我只负责点确认窗口”
- 你应调用本 Skill

### 示例 3：连续两轮没定位
- 用户说：“已经改了两次还是不对，别再猜了”
- 你应调用本 Skill

## 成功标准

- 每轮都有明确目标
- 每轮都有证据
- 每轮都有结论
- 连续失败后能自动转入插桩
- 用户参与被压缩到必要最小范围
