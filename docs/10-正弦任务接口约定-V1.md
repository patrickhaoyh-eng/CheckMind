# 正弦任务接口约定 V1

## 1. 目的

- 本文用于固定“创建核验任务”时前端传给后端的最小参数契约。
- 当前阶段以前端不迭代、后端先跑通为原则；前端后续联调时直接以本文为准。
- 本文只覆盖正弦任务模板里已经明确的 `Notch Profiles` 数量输入，不扩展随机任务。

## 2. 适用范围

- 任务类型：`sine`
- 任务场景：创建正弦核验任务
- 当前必须支持的能力：由前端传入 `notchProfileCount`，后端展开为 `1..N`

## 3. 前后端契约

### 3.1 前端提交字段

| 字段名 | 类型 | 必填 | 含义 | 示例 |
|------|------|------|------|------|
| `taskType` | `string` | 是 | 任务模板类型 | `sine` |
| `notchProfileCount` | `int` | 正弦任务时必填 | `Sine Setup` 页面下 `Notch Profiles` 个数 | `5` |

### 3.2 当前 V1 约束

- `taskType = sine` 时，`notchProfileCount` 必填。
- `notchProfileCount` 必须是正整数，最小值 `1`。
- V1 不支持前端直接传离散序号列表，例如 `1,3,5`。
- V1 不支持前端直接传环境变量名；前端只传业务字段。

## 4. 请求示例

```json
{
  "taskType": "sine",
  "notchProfileCount": 5
}
```

## 5. 后端语义

### 5.1 字段映射

- 前端字段：`notchProfileCount`
- 运行时环境变量：`CHECKMIND_NOTCH_PROFILE_COUNT`
- 执行层展开结果：`1..N`

例如：

- `notchProfileCount = 1` -> 执行序号 `1`
- `notchProfileCount = 2` -> 执行序号 `1,2`
- `notchProfileCount = 5` -> 执行序号 `1,2,3,4,5`

### 5.2 当前解析优先级

执行层当前兼容三种口径，优先级如下：

1. `CHECKMIND_NOTCH_PROFILE_INDEXES`
2. `CHECKMIND_NOTCH_PROFILE_INDEX`
3. `CHECKMIND_NOTCH_PROFILE_COUNT` / `notchProfileCount`

说明：

- 第 1、2 项仅作为现有调试脚本兼容入口保留。
- 正式任务创建接口一律以 `notchProfileCount` 为主口径。

## 6. 后端校验规则

### 6.1 入参校验

- 当 `taskType != sine` 时，不解析 `notchProfileCount`。
- 当 `taskType = sine` 且缺少 `notchProfileCount` 时，应返回参数错误。
- 当 `notchProfileCount <= 0` 时，应返回参数错误。
- 当 `notchProfileCount` 不是整数时，应返回参数错误。

### 6.2 运行时失败口径

若 `notchProfileCount = N`，则执行层按 `1..N` 顺序处理；任一序号失败即 fail-fast。

建议错误文案按可行动方式输出，例如：

- `Notch Profile #3 打开失败`
- `Notch Profile #4 行点击点超出列表 ROI`
- `Notch Profile #2 关闭后未回到列表`

### 6.3 运行时数量不符口径

- 若前端传入 `notchProfileCount = N`，但执行层在处理第 `K` 个目标序号时，连续两次点击后仍停留在上一条有效列表项，则判定“实际列表数量小于用户输入数量”。
- 该场景不再重复打开上一条 `Notch Profile` 内容，也不把它当成新的成功项。
- 执行层应按已完成的实际序号正常结束，并在运行结果中写出 `NotchProfileCountMismatch`。
- 运行脚本 / 前端联调阶段应向用户明确提示：`Notch Profile Count` 与实际列表数量不符，请检查。

当前正式验证基线：

- run `2523cbe57bb44a3188258345fe02e987` 已确认当用户输入 `NotchProfileCount=3`、实际仅有 2 条列表项时：
  - 不再重复打开第 2 条旧内容
  - `testlab_run.json` 正确写出 `NotchProfileCountMismatch`
  - 终端正确输出 `last_run=...` 与 `notch_profile_count_mismatch=...`
  - 脚本正确弹出中文确认提示框

## 7. 前端展示建议

### 7.1 表单字段

- 展示名称：`Notch Profiles 个数`
- 说明文案：`请填写 Sine Setup 界面下 Notch Profiles 的个数`
- 控件类型：数字输入框

### 7.2 交互建议

- 必填校验在前端先做一次。
- 仅允许输入正整数。
- 不要让用户手动输入 `1,2,3` 这类序号串。

## 8. 联调说明

- 当前阶段允许后端与脚本直接赋值验证，不要求前端页面已完成。
- 前端后续接入时，只需把 `notchProfileCount` 传给后端即可。
- 后端不应要求前端感知 `CHECKMIND_NOTCH_PROFILE_COUNT` 这类运行时细节。
- 若收到运行结果中的 `NotchProfileCountMismatch`，前端应把该次任务标记为“已执行但配置待核实”，并向用户展示数量不符提示，不要误判为“成功处理了全部 `1..N`”。

## 9. 当前实现对照

- 后端解析器：`CheckMind.App/Core/NotchProfileIndexResolver.cs`
- 正式 runner 使用：`CheckMind.App/Core/TestlabAutomationRunner.cs`
- 本机验证入口：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run_testlab_with_notch_profiles.ps1 -NotchProfileCount 5
```

## 10. V1 非目标

- 随机任务模板的参数定义
- 前端页面开发与联调细节
- 离散序号列表输入，例如 `1,3,5`
- 自动从 UI 内容识别 `Notch Profiles` 实际行数
