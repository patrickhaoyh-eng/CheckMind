# 人工标定统一基线开发计划 V1

## 1. 目标
- 在不改变“`PgUp/PgDn` 作为执行层动作”这一事实的前提下，把项目切换到“人工标定驱动”的长期基线。
- 本计划只定义开发顺序、验收口径和迁移边界，不直接代表代码已完成。
- 本计划从属于 `docs/08-人工标定统一基线.md` 与项目级质量约束；实现顺序允许按实际收敛调整，但质量门禁、证据要求、fail-fast、固定工位边界与非目标不变。
- 当前进展更新：
  - `Channel Setup` / `Sine Setup` 已按固定截图框稳定产出 evidence
  - 固定工位门禁（按屏规格）已接入并通过实机验证
  - `Sine Setup` 页签验真签名与 `top_serial` 已完成主链路收口
  - `top_serial` 已明确固化为“样本生成时机必须与正式运行一致”的主口径
  - 实际推进顺序已发生偏移：为优先收口主链路可用性，`Phase 4` 的核心内容已先于 `Phase 3` 跑通并固化
  - `Task B` 已补入 `table_evidence_*.json` 结构化报告，用于固化拼接/去重与 `coverage.json`、`table_scroll_events_*.json` 的一致性校验
  - 当前下一优先级已调整为“正弦振动试验素材与人工标定对象补齐”，`Phase 5` 暂后，不在素材与入口策略未收口时提前清旧基线

## 1.1 不变约束
- 不改变顶层质量约束：证据优先、可追溯、fail-fast、停止猜测后转插桩。
- 不改变治理边界：人工标定、统一核验、固定工位门禁、结构化 profile 主口径。
- 不改变非目标：不扩展自动改参/自动点击修改/自动阻断试验，不回退到 OCR 主链路，不做跨电脑自适应承诺。

## 1.2 可调整项
- 可根据实机问题收敛情况调整阶段先后顺序。
- 可优先处理影响主链路真实性、证据完整性和口径一致性的收口项。
- 可在保留兼容兜底的前提下，逐步收敛旧字段、旧脚本和旧治理路径。

## 2. 计划范围
- 纳入本轮：
  - profile 扩展：支持截图框、弹窗点击点、更多验真框
  - 运行前统一核验
  - 固定工位门禁（按屏规格 `1920×1080/100%`，并启用 Per-Monitor V2 DPI）
  - 前端重标入口
  - `Channel Setup` / `Sine Setup` 截图框接入
  - 旧动态截图区域策略的迁出计划
- 不纳入本轮：
  - 跨电脑/跨分辨率自适应
  - OCR 回归主链路
  - 采集阶段的智能减图优化

## 3. 设计分层

### 3.1 治理层
- 标定对象定义
- profile 持久化
- 运行前统一核验
- 核验失败阻断
- 前端一键重标

### 3.2 执行层
- 在核验通过的前提下执行：
  - 页签点击
  - 弹窗点击
  - 验真截图
  - 表格截图
  - `PgUp/PgDn` 滑窗取证

## 4. 分阶段实施

### Phase 1：数据模型与配置基线
**目标**
- 让 profile 能表达“人工标定统一基线”所需对象。

**任务**
- 定义统一标定对象模型：
  - `ClickPoint`
  - `CaptureRoi`
  - `VerifyRoi`
  - `DialogActionPoint`
- 扩展当前固定工位 profile 文件结构
- 为 `Channel Setup`、`Sine Setup` 增加截图框配置位
- 为后续弹窗点击点预留结构

**验收标准**
- profile 可同时保存页签点击点、截图框、验真框
- 旧配置可迁移或兼容读取
- 结构命名足够稳定，避免后续再改一轮

### Phase 2：运行前统一核验
**目标**
- 在正式执行前统一验证所有已标定对象是否可用。

**任务**
- 增加统一核验入口
- 逐项核验：
  - 固定工位门禁（按屏规格，不按第几屏编号）：
    - Testlab 窗口所在显示器分辨率必须为 `1920×1080`
    - Windows 缩放必须为 `100%`
    - 进程启用 Per-Monitor V2 DPI 感知，避免混合 DPI 造成坐标虚拟化
  - 页签点击点
  - 验真 ROI / 签名
  - 截图 ROI 是否在有效窗口范围内
  - 必要时核验弹窗点击点
- 输出结构化核验结果文件
- 任一失败直接阻断执行

**验收标准**
- 核验结果可以结构化落盘
- 执行前可以看到“通过/失败/失败项”
- 失败时不进入正式抓取
- 固定工位门禁失败时有可行动建议（移动窗口到 1080p/100% 显示器、调整缩放/分辨率等）

### Phase 3：前端重标入口
**目标**
- 把“重新标定”从脚本式补丁变成正式前端能力。

**任务**
- 在前端增加“统一核验”入口
- 在前端增加“重新标定”入口
- 支持按对象类型重标：
  - 页签点击点
  - 截图框
  - 验真框
  - 弹窗点击点
- 核验失败后支持直接跳转重标

**验收标准**
- 用户不需要改脚本即可完成核验与重标
- 重标完成后 profile 能立即生效
- 核验失败提示可行动

### Phase 4：执行链路接入
**目标**
- 让执行层全部切换到“按标定对象执行”。

**任务**
- 切页动作改为只读取标定点击点
- 验真改为只读取标定验真 ROI / 签名
- `Channel Setup` / `Sine Setup` 截图改为只读取标定截图框
- 保留 `PgUp/PgDn` 作为滑窗执行动作
- 保留“每次 `PgDn` 后立刻截一屏”

**验收标准**
- 不再依赖运行时动态猜截图框
- 证据图来源可追溯到具体标定对象
- 两个页面都能按固定截图框稳定产出 evidence
- 已补充收口口径：
  - 运行时优先读取结构化 `VerifyTargets`
  - `Sine Setup` 页签验真签名应稳定一次命中
  - `top_serial` 样本必须由“聚焦表格 + PgUp 回顶稳定后”的同口径帧生成

### Phase 5：旧治理基线迁出
**目标**
- 删除或封存不再使用的旧路线，避免后续维护分叉。

**任务**
- 标记并移除动态截图区域推导逻辑
- 标记并移除与截图位置判定有关的 OCR 旧路径
- 清理只服务旧治理基线的环境变量与注释
- 更新脚本与文档，统一指向新基线

**验收标准**
- 文档、脚本、实现不再出现互相冲突的基线表述
- 新同事只看当前文档即可理解主路径

## 5. 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| profile 结构扩展破坏旧读取 | 高 | 先做兼容读取，再逐步迁移 |
| 前端重标入口与现有脚本路径并存导致混乱 | 高 | 先定义单一入口，再逐步收敛脚本用途 |
| 截图框人工标定后仍漏行 | 高 | 坚持覆盖优先，保留 `PgDn` 每次截屏与后处理去重 |
| 旧动态逻辑残留导致实际运行混用新旧基线 | 高 | 在 Phase 5 做显式迁出清理 |

## 7. 当前专项：正弦振动试验素材补齐

### 7.1 收口目标
- 把正弦振动试验相关的全部抓取对象统一纳入一套“抓取通用范式与差异化配置项表”。
- 该表既包含新增待补素材，也包含已经做好的 `Channel Setup` / `Sine Setup`，避免形成新旧两套抓取体系。
- 后续实现遵循“补层不换底”，先兼容式扩展，不破坏当前已跑通的 `Channel Setup` / `Sine Setup` 主链路。

### 7.2 `Notch Profiles` 变量行处理策略
- `Notch Profiles` 的 `1..N` 列表是后续待抓取素材的入口，不按“每个序号单独人工标定”处理。
- 运行时按“目标序号 -> 目标行 -> 点选该行 -> 点击 `Edit` -> 进入编辑界面”的导航链执行。
- `Notch Profile` 页面下的执行不是单次入口，而是由 `Notch Profiles` 列表序号驱动的循环：
  - 在 `Notch Profiles` 列表中按目标序号逐行进入
  - 每次点选一个序号后，点击 `Edit`
  - 进入对应 `Notch Profile` 页面执行抓取 / 核验 / 关闭动作
  - 返回后继续处理下一个序号
  - 直到当前任务要求的全部序号完成
- 人工标定主口径固定的是：
  - 列表容器 ROI
  - 首行基准
  - 行高 / 行步进
  - `Edit` 按钮入口
- 风险边界：
  - 用户若调整行高、列宽或列表布局，会破坏行定位规则。
  - 该场景不做运行时自适应兼容，必须纳入固定工位 / preflight 约束；布局漂移时直接阻断。

### 7.3 子窗口抓取通用范式
- 统一流程：打开子窗口 -> 强制最大化 -> 可选切换页签 / 列表定位 -> 固定区域截图或 `PgUp/PgDn` 滑窗取证 -> 关闭窗口。
- 对可调整窗口，一律先执行 `Alt+Space` + `X` 强制最大化，再复用固定坐标。
- 对需要翻页的子窗口，继续复用已经在 `Channel Setup` / `Sine Setup` 验证通过的 `PgUp/PgDn + 拼接 + 去重` 能力；后续实现以函数复用为方向，不再为每个窗口单独造轮子。
- 关闭窗口优先复用“最大化后右上角关闭按钮”统一坐标；仅在窗口类型证明位置不稳定时才单独标定。
